## Week 1: コード構造の土台整備

### 目的
CRMView.jsx (1003行) に全機能が混在している状態を解消。以後の機能追加でリグレッションが起きない構造にする。

### 作業項目
- [ ] `CRMView.jsx` を以下に分割
  - `CRMView.jsx`（オーケストレーション役、~150行）
  - `CRMTable.jsx`（タブ＋テーブル本体）
  - `CRMTableRow.jsx`（1行分の表示ロジック・色判定）
  - `ClientFormModal.jsx`（新規・編集モーダル）
  - `RewardDetailModal.jsx`（報酬詳細モーダル）
  - `ContactDrawer.jsx`（既に分離済みの場合はそのまま）
- [ ] props drilling 解消: `clientData` `setClientData` `contactsByClient` 等を Context で共有
- [ ] 最終接点取得を SWR / React Query でキャッシュ化（5分間）
- [ ] 動作確認: 既存タブ・ソート・編集・新規追加が全て無傷で動くこと

### 完了条件
- 既存機能が全て同じ操作感で動く
- 各ファイルが300行以下に収まる
- 開発者がCRMの任意の機能を2分以内に該当ファイルにたどり着ける

## Week 6: CRM新規開拓ボードのフル機能化（コピペ方式）

### 目的
v1の最小実装から、既存ソーシング架電と同等のフル機能をCRM新規開拓側にも提供する。
**既存ソーシング架電のコードには一切手を入れず、コピペで CRMLead* ファイル群を作成・拡張する。**

### 引き算する機能（CRM新規開拓には不要）
1. リスト複製
2. クライアント紐付け
3. PDFスクリプト
4. 都道府県フィルタ
5. ライブ進捗共有
6. ピン留め／一時保存
7. AI ロープレ分析

### 残す（フル機能から外さない）
範囲指定 / アポ獲得詳細モーダル / オートコール / Q&Aタブ / 集中モード /
録音再生 / 最小化 / 検索・ソート / F1〜F8ショートカット / 再コールモーダル /
ヘルプ画面 / アーカイブ など

### 6-A: 一覧型架電画面の機能拡張（CRMLeadCallingScreen）
- [ ] 範囲指定（No.50〜100だけ架電）
- [ ] 検索（企業名フィルタ）
- [ ] ソート（最終発信日・架電状況）
- [ ] F1〜F8 ショートカット（既存と同じキー配置）
- [ ] 最小化／PiP
- [ ] ショートカット一覧ヘルプ（`?` キー）
- [ ] 録音URL保存＆再生

### 6-B: 集中モード（CRMLeadCallFlowView）
- [ ] 既存 CallFlowView をベースに新規ファイル作成
- [ ] 1社フォーカスUI（左に企業リスト、中央に詳細＋スクリプト、右に履歴）
- [ ] 大型ステータスボタン
- [ ] スクリプト並列表示
- [ ] オートコール（次企業への自動発信）
- [ ] Q&Aタブ
- [ ] 録音再生

### 6-C: 専用モーダル
- [ ] アポ獲得詳細モーダル（面談日時・担当者名・先方所感を入力。CRM転記時のクライアント情報にも反映）
- [ ] 再コールモーダル（受付/キーマン再コール時の予約日時入力）

### 6-D: リスト一覧画面の機能拡張（CRMLeadGenView）
- [ ] アーカイブ機能（古いリストを非表示化）
- [ ] リスト名・業界の編集
- [ ] リスト並び替え（取込日／業界）
