## ゴール

社内SpanaviのCRMページを「数億円レベルのSaaS」に引き上げ、その上で新たに「新規クライアント開拓ボード」（CRMサブビュー）を実装する。

## Week 1: コード構造の土台整備

### 目的
CRMView.jsx (1003行) に全機能が混在している状態を解消。以後の機能追加でリグレッションが起きない構造にする。

### 作業項目
- [ ] `CRMView.jsx` を以下に分割
  - `CRMView.jsx`（オーケストレーション役、~150行）
  - `CRMTable.jsx`（タブ＋テーブル本体）
  - `CRMTableRow.jsx`（1行分の表示ロジック・色判定）
  - `ClientFormModal.jsx`（新規・編集モーダル）
  - `RewardDetailModal.jsx`（報酬詳細モーダル）
  - `ContactDrawer.jsx`（既に分離済みの場合はそのまま）
- [ ] props drilling 解消: `clientData` `setClientData` `contactsByClient` 等を Context で共有
- [ ] 最終接点取得を SWR / React Query でキャッシュ化（5分間）
- [ ] 動作確認: 既存タブ・ソート・編集・新規追加が全て無傷で動くこと

### 完了条件
- 既存機能が全て同じ操作感で動く
- 各ファイルが300行以下に収まる
- 開発者がCRMの任意の機能を2分以内に該当ファイルにたどり着ける

## Week 2: 情報設計＋抜け漏れ防止

### 2-A. KPIダッシュボード（CRMページ最上部）
- [ ] 4枚のカード: 今月成約数 / 累計売上 / 目標対比% / ステータス別件数
- [ ] 前月比の増減を矢印＋数字で表示
- [ ] クリックで詳細フィルタへ遷移

### 2-B. テーブル列の再設計
- [ ] 列を再優先度づけ: ステータス → 企業名 → 最終接点 → 主担当 → **次回接点予定日（新設）** → **目標対比%（新設）**
- [ ] 「次のアクション」列を追加（自動判定で「メール送信待ち」「商談化フォロー必要」等を表示）
- [ ] 業界・メモ等は詳細ページに移動

### 2-C. フォロー漏れアラート
- [ ] 右上に「フォロー漏れ XX件」赤バッジ。クリックで該当行のみ表示
- [ ] 「予定日超過 XX件」バッジも同様（面談予定タブ用）

### 2-D. 優先度スコア
- [ ] スコア計算式を実装（報酬額 × 目標達成率 × 経過日数の重み付け）
- [ ] 各行に1〜100のスコアを赤/黄/灰バッジで表示
