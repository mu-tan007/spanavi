# スパキャリ事業 全体仕様書

> 本ドキュメントは、スパナビ内「スパキャリ」事業の運営ダッシュボードおよびクライアントポータルを再構築するための全体仕様書である。既存スパキャリ向けコード（applications / deals_career / members_career 等）は全廃棄し、本仕様に基づき新規構築する。

- ステータス：壁打ち確定済み、実装フェーズ着手前
- 想定読者：基盤構築および並列実装を担当する Claude Code エージェント

## 0. プロジェクトサマリ

### 0.1 目的

M&Aソーシングパートナーズが提供するキャリアコーチングサービス「スパキャリ」の運営を、社内ツール「スパナビ」内で完結させる。受講生（顧客）向けポータルと、運営／トレーナー向けダッシュボードを構築する。

### 0.2 スコープ

- 運営ダッシュボード 8画面
- クライアントポータル 5画面
- AI機能 5種（議事録 / 30項目生成 / 強み診断 / フレーズ抽出 / 今日のひとこと）
- 外部連携（Slack / 動画ホスティング / Zoom URL）
- データモデル
- 業務フロー

### 0.3 スコープ外

- 既存スパナビ全体の事業切替UI（既存仕組みに乗る）
- 既存ソーシング事業
- マーケティングサイト
- 課金・契約管理（外部システム）

## 1. 用語定義

| 用語 | 意味 |
|---|---|
| スパキャリ | M&Aソーシングパートナーズが提供するキャリアコーチングサービス |
| スパナビ | スパキャリ運営用社内ツール（顧客管理・事前課題・議事録・直案件DB 等） |
| トレーナー | お客様を担当するコーチ。営業＝コーチであり、契約後の引継ぎは行わない |
| 運営 | 管理者ロール。全権限を保持 |
| 受講生 | スパキャリを受講するお客様。クライアントポータルのみアクセス可能 |
| ソーシャルスタイル診断 | 4タイプ診断（論理分析型・行動推進型・感情表現型・協調共感型） |
| 強み診断 | 第2回事前課題タイミングで受講する独自診断。プロフィールに反映 |
| Slackゲストチャンネル | 顧客と唯一連絡を取るチャンネル。フルネーム漢字で命名（例：山田太郎） |
| セッション | 第0回（キックオフ）〜第8回 までの全9回 |
| キックオフ | 第0回セッション。ヒアリングシートに沿って今後の流れを確認する |
| 事前課題 | 各セッション前に受講生が回答する課題。第1回〜第8回前の全8サイクル |

### 運用上の鉄則

- 顧客との連絡はSlackゲストチャンネルのみで行う
- セッションは原則 週1回・連続実施
- セッションはZoomで必ず録画し、スパナビに保管する
- 第3回セッション終了時点が全額返金保証の期限カットオフ
- スパキャリ UI 内で絵文字は使用しない（金融プロフェッショナル感の統一）

### 4.2 重要な不変条件

- 進捗率は **第0〜8回の全9セッション基準**（例：第3回終了＝4/9＝44%）
- 第0回（キックオフ）には事前課題なし
- 事前課題は **第1回〜第8回前の全8サイクル**
- セッション完了時、必須ゲート：第0回はヒアリングシート全項目チェック、第1〜8回は議事録＋ヒアリングシート添付
- 第4回突入で直案件DB閲覧権限が**自動付与**（手動切替なし）

## 5. 業務フロー全体

### 5.1 契約直後〜診断〜アカウント生成

```
1. 契約・入金完了
2. Slackゲストチャンネルを開設（運営手動、フルネーム漢字で命名）
3. 診断URLをSlackで送付（招待時アカウント未作成）
4. 受講生がスパナビ内でソーシャルスタイル診断に回答（中断・再開可）
5. 診断完了 → 顧客ページ自動生成 + スパナビアカウント自動生成
6. スパナビ権限付与通知が自動でSlackに送付（メール＋初期パスワード含む）
7. 運営が顧客一覧の「要対応（未アサイン）」から手動でトレーナーをアサイン
8. 受講生が初回ログイン時に強制パスワード変更
```

### 5.2 キックオフ（第0回）

```
1. キックオフミーティング実施（Zoom録画あり）
2. トレーナーがヒアリングシート全項目（4.3.1〜4.3.9）を口頭確認しチェック
3. 第1〜第8回の大枠日程をその場で入力（カレンダー形式）
4. 第1回セッション開始日時を確定（日付＋時間）
5. 動画アップロード → AI議事録生成
6. 「完了」ボタン押下
   - 第1回事前課題テンプレート（共通項目）が自動配布
   - Slack通知（クライアントポータルに反映）
```

### 5.3 第1回〜第8回セッション運用（標準フロー）

```
（セッション当日）
1. セッションをZoomで実施・録画
2. セッション動画をスパナビにアップロード
3. AIが議事録を自動生成
4. ヒアリングシート全項目をチェック
5. 「セッション完了」ボタン押下
   - ステータス自動進行（次回セッションへ）
   - クライアントポータルに議事録反映
   - 満足度アンケート通知をSlack送信（自動）
   - AIが次回事前課題30項目を自動生成（ドラフト）

（当日〜翌日中）
6. トレーナーが30項目ドラフトを手動修正・追加・削除
7. 「完了・通知」ボタン押下
   - 事前課題が確定し、クライアントポータル公開
   - Slackに「クライアントポータルに反映」通知（自動）
   - 提出期限が次回セッションの3日前に自動設定

（次回セッションまで）
8. 受講生がスパナビ上で事前課題に回答（部分提出可）
9. 締切3日前で未提出/部分提出なら「要対応」アラート
10. トレーナーが提出項目を順次「OK」判定
11. 全項目OKで該当回の事前課題完了

（次回セッション開始時間の確定）
12. 前回セッションの最後に次回開始時間を決定 → スパナビへ記入
```

### 5.4 第4回突入時の権限切替

- 第3回「完了」押下 → 自動で第4回「次回実施」化
- 第4回セッションを開始した瞬間に直案件DB閲覧権限が自動付与
- 第4回内で直案件DBが閲覧できることをお客様と一緒に確認

### 5.5 卒業

```
1. 第8回セッション完了ボタン押下
2. 自動的に卒業/アフターサポートへ移行（手動承認なし）
3. アフターサポート権限（月2回チャット／AI講座継続／AIコミュニティ／スパナビライセンス継続）が付与
```

## 6. クライアントポータル画面仕様

### 6.1 基本情報（マイページ）

参考画像：イメージ⑦

#### レイアウト

- ヘッダー：「マイページ」タイトル＋サブコピー
- 中央上部：原動力フレーズエリア
  - 第1回事前課題から自動抽出された「自分に伝えたいフレーズ」
  - 「あなたの原動力」バッジ＋ビジュアル背景
- 中央中段：3カラム
  - 基本情報カード（編集ボタン付き）
  - 学習の進捗カード（AI講座視聴の円グラフ）
  - セッションの進捗カード（次セッション準備状況）
- 中央下部：あなたの目標カード×3（事前課題から自動引用）
  - 今回のスパキャリで絶対に手に入れたいもの
  - お金以外で本当に達成したい価値観
  - 尊敬している人物とその理由
- 右カラム3エリア
  - 次回セッション情報＋事前にやっておくこと
  - 直近のセッション感想（未入力なら誘導）
  - 「今日のひとこと」モチベーショナルメッセージ
- 左バナー下部：困ったときは／よくある質問／お問い合わせ／サポートに連絡／ログアウト

#### 編集権限

- 編集可：ニックネーム（呼ばれたい名前）、プロフィール画像
- 編集不可：氏名／メールアドレス／電話／生年月日／職業／スパキャリ開始日／次回セッション

#### データソース

- 原動力フレーズ：`spacareer_assignments_task_items`（第1回事前課題の特定項目）からAI抽出
- 目標3カード：第1回事前課題の所定3項目から自動引用
- 今日のひとこと：AIが受講生情報＋過去事前課題からパーソナライズして生成（日次バッチ）

### 6.2 事前課題

参考画像：イメージ③

#### レイアウト

- 左バナー：5メニュー固定
- 中央：設問リスト（縦並び）
  - 設問ごとに「必須」マーク（基本的に全項目必須）
  - テキスト入力欄に文字数カウンター
  - ファイル添付欄（最大3ファイル、形式制限なし、1ファイル50MB）
- 下部固定バー：「一時保存」「全ての回答を保存」「回答を提出」の3ボタン
- 右カラム：回答進捗（円グラフ）＋設問一覧（クリックで該当設問にジャンプ）

#### 挙動

- 自動保存なし、一時保存ボタンによる手動保存のみ
- 部分提出（書きかけ）許容
- 1項目でも未回答があれば全体ステータスは「部分提出」
- 締切3日前以降で部分提出または未提出は「要対応」アラート対象

#### ステータス遷移

| ステータス | 意味 |
|---|---|
| 未通知 | トレーナーが「完了・通知」未押下 |
| 未提出 | 配信済み、受講生が一度も回答していない |
| 部分提出 | 一部のみ提出（1項目でも未回答あり） |
| 提出済み | 全項目100%提出完了 |
| 完了 | トレーナーが全項目OK判定 |

### 6.3 セッション感想

### 6.4 AI講座

参考画像：イメージ①

#### レイアウト

- ヘッダー：学習進捗パネル（視聴済み／視聴中／未視聴の本数＋円グラフ）
- フィルタタブ：すべて／未視聴／視聴中／視聴済み／お気に入り
- カテゴリ別グループ表示
  - はじめに見てほしい動画
  - 基礎スキル編
  - プロンプト基礎
  - プロンプト実践
  - 情報収集
  - 文章作成
  - アイデア発想
  - 資料作成
  - 業務効率化
  - ※カテゴリ自体は運営が自由追加・編集可能
- 動画カード：サムネ／タイトル／所要時間／カテゴリバッジ／視聴ステータス／お気に入りボタン

#### 視聴済み判定

- 動画の80%以上を再生で「視聴済み」
- 0% = 未視聴、1〜79% = 視聴中、80%以上 = 視聴済み

#### お気に入り

- 受講生が個人的にマーク可能
- データは受講生アカウントに紐づく
- 運営側分析（お気に入り数ランキング）でアップロード選定に活用

  - 全体進捗（進行中顧客数）
  - 完了（全セッション完了顧客数）
  - 遅延アラート（事前課題未通知・期限超過）
  - 次回セッション予定（今週予定数）

#### 各回タブの一覧表示

| 列 | 内容 |
|---|---|
| 顧客名 | 名前＋ID |
| セッション実施日 | 予定日 or 実施日 |
| ステータス | 未実施／完了／次回実施 の3種 |
| 事前課題の状態 | 未通知／未提出／部分提出／提出済み／完了 |
| アラート | 該当する要対応アラート（あれば） |
| 担当コーチ | トレーナー名 |

- ソート：セッション実施日昇順
- 「実施中」ステータスは廃止
- 「未実施」「通知済み」廃止により「未提出」または「通知済み」に統合

#### 「次回実施」判定ロジック

- **直近の完了セッションの次の番号**を「次回実施」として自動マーク
- 実施日未確定でもマーク可（実施日が決まったら昇順に反映）

#### 完了フロー（必須ゲート＋自動連鎖）

```
1. セッション動画をアップロード（管理画面）
2. AIが議事録自動生成
3. 議事録＋ヒアリングシート両添付の確認
4. ヒアリングシート全項目チェック（必須ゲート）
5. 「セッション完了」ボタン押下
   ├─ ステータス自動進行（次回セッションへ）
   ├─ 議事録がクライアントポータルに反映
   ├─ 満足度アンケート通知をSlack送信（自動）
   └─ AIが次回事前課題30項目を自動生成（ドラフト）
6. トレーナーが30項目を手動修正
7. 「完了・通知」ボタン押下
   ├─ 事前課題がクライアントポータル公開
   ├─ Slack通知（クライアントポータルに反映）
   └─ 提出期限を次回セッションの3日前に自動設定
```

#### 第0回（キックオフ）の特殊扱い

| 項目 | 第0回 | 第1〜8回 |
|---|---|---|
| 事前課題 | なし | あり |
| 完了の必須ゲート | ヒアリングシートの全チェック完了 | 議事録＋ヒアリングシート添付 |
| 完了押下後の挙動 | 第1回事前課題（共通テンプレ）自動配布 | AIが次回事前課題30項目を自動生成 |

#### 「通知」ボタンの自動文面切替

- 事前課題が未着手の場合：事前課題リマインドテンプレ
- 締切当日の場合：締切リマインドテンプレ

#### 卒業フロー

- 第8回完了ボタン押下 → 自動で卒業/アフターサポート移行（手動承認なし）

## 8. AI機能仕様

### 8.1 AI議事録自動生成

- トリガー：セッション動画アップロード完了時
- 入力：動画ファイル（音声）
- 処理：音声→テキスト変換（Whisper等）→ 要約・整形（Claude）
- 出力：構造化された議事録（PDF/テキスト両対応）
- 反映先：個人ページ「セッション履歴」タブ＋クライアントポータル「セッション履歴」タブ（運営完了押下後）

### 8.2 AI事前課題30項目自動生成

- トリガー：セッション完了ボタン押下時（第1〜7回完了時、次回分を生成）
- 入力：当該セッションの議事録／ヒアリングシート／過去事前課題／顧客プロフィール／ソーシャルスタイル診断結果／強み診断結果
- 処理：Claudeに対し「次回事前課題として30項目を生成せよ」とプロンプト実行
- 出力：30項目のドラフト（タイトル＋設問本文＋必須/任意マーク）
- フロー：トレーナーが手動で追加・修正・削除→「完了・通知」で確定

> AIプロンプトは運営のみ編集可（テンプレート管理）

### 8.3 ソーシャルスタイル診断

- 質問項目：30問（Claudeが論文・公表データに基づき素案作成）
- 中断・再開：最後の質問位置を保存、URLで再開
- 完了時挙動：4タイプ判定＋スコアバランス算出→ 顧客ページ自動生成＋スパナビアカウント自動生成→ 運営に「未アサイン新規顧客」通知

### 8.4 強み診断

- 受講タイミング：第2回事前課題の1項目として実施
- 質問項目数：TODO（実装フェーズで素案提示）
- 出力：強み・価値観のテキスト＋スコアバランス
- 反映先：個人ページ「強み・価値観」タブ／クライアントポータル基本情報タブ

### 8.5 フレーズ抽出（あなたの原動力）

- トリガー：第1回事前課題の提出完了時
- 入力：第1回事前課題の所定項目（生い立ち／動機／ゴール設計）
- 処理：Claudeに「自分に伝えたいフレーズ」を1〜2文で抽出
- 出力：マイページのヒーローエリアに表示するフレーズ

### 8.6 今日のひとこと

- トリガー：日次バッチ
- 入力：受講生プロフィール／過去事前課題／直近のセッション議事録
- 処理：Claudeに「今日のひとこと」をパーソナライズ生成
- 出力：マイページ右下に表示
